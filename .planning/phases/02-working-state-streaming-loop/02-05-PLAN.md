Purpose: 将当前“Windows Terminal 可用、VS Code Terminal 仍有偏移”的状态收口为“至少具备明确兼容模式与稳定输入体验”，避免用户只能切换终端才能正常使用。
Output: VS Code Terminal 环境识别、兼容模式下的简化底部输入布局、专项回归测试，以及更新后的本地验收说明。
</objective>

- truth: 兼容模式优先保证输入稳定性，其次才是视觉完整度。
- truth: 用户需要明确知道 Windows Terminal 是通过环境，VS Code Terminal 是否启用兼容模式与还有哪些限制。
</must_haves>

<task>
  <name>Task 2: 为 VS Code Terminal 提供更稳定的兼容输入布局</name>
  <files>src/app.tsx, src/components/Composer.tsx, src/components/WelcomeCard.tsx, tests/components/composer.test.tsx, tests/components/app-command-mode.test.tsx, tests/components/app-vscode-terminal-compat.test.tsx</files>
  <action>
    为 VS Code Terminal 设计兼容模式：
    1. 简化底部输入区样式，优先使用单行 prompt 风格，减少边框、动态高度和靠近输入区的高频重排。
    2. 保证命令候选面板与输入区之间的相对位置稳定，避免候选区域挤压输入光标锚点。
    3. 必要时对欢迎卡片或底部布局做轻量降级，但不能影响正常提问、Working、Streaming、`/model`、`/exit` 和 `Esc` 中断。
    用测试锁定兼容模式仍保留命令模式和输入提交能力。
  </action>
  <verify>npm test -- tests/components/composer.test.tsx tests/components/app-command-mode.test.tsx tests/components/app-vscode-terminal-compat.test.tsx</verify>
  <acceptance>
    - VS Code Terminal 兼容模式使用更稳定的输入区布局
    - `/`、`/model`、`/exit`、普通提问和 `Esc` 中断能力不回退
    - 普通终端默认布局不受影响
  </acceptance>
