## 输入区行为

| Option | Description | Selected |
|--------|-------------|----------|
| 单行优先 | `Enter` 直接提交，先把 REPL 主路径做顺 | ✅ |
| 多行优先 | 一开始就支持复杂多行编辑，但会增加终端输入复杂度 | |
| 混合模式 | 默认单行，某些快捷键进入多行模式 | |

**User's choice:** `[auto]` 选择“单行优先”（推荐默认）
**Notes:** 本阶段重点是底部固定输入区与命令模式，不把多行输入复杂度提前引入。

## 命令面板

| Option | Description | Selected |
|--------|-------------|----------|
| 列表 + 说明 | 每个命令显示名称和一句说明，便于发现 | ✅ |
| 仅显示命令名 | 更简洁，但新用户理解成本更高 | |
| 详情面板模式 | 右侧或下方显示更多描述，但会增加布局复杂度 | |

**User's choice:** `[auto]` 选择“列表 + 说明”（推荐默认）
**Notes:** 与截图目标一致，命令模式必须是可发现的，而不是记忆型功能。

## 键盘交互

| Option | Description | Selected |
|--------|-------------|----------|
| 标准 CLI 键位 | `Up/Down` 高亮，`Enter` 选择，`Tab` 补全，`Esc` 退出 | ✅ |
| 精简键位 | 只支持上下和回车，实现最少 | |
| 激进快捷键 | 增加更多单键行为，但学习成本更高 | |

**User's choice:** `[auto]` 选择“标准 CLI 键位”（推荐默认）
**Notes:** 与产品截图和需求文档保持一致，降低认知成本。
